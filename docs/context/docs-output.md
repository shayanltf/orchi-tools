# Docs Output

Use when writing or editing non-code files.

## Contract

- Output finished document only.
- No preamble.
- No "as requested" note.
- No change summary inside file.
- No inline diff markers.
- No version-story block unless the file is a changelog.
- Preserve existing voice when editing.
- Fold changes into file as if originally written that way.
- Code files are exempt; code review can use diffs.

## Practical Check

- README explains current project, not edit history.
- INDEX links exact files and says when to read each one.
- Context files are reusable agent context, not task logs.
- Changelog records release history only.
- Comments and issue reports carry process summary, not repo docs.
